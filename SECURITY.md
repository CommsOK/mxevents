# Security Policy

## Supported Versions

This project currently supports security fixes for the latest released version.

## Reporting a Vulnerability

Please **do not** open a public GitHub issue to report a security vulnerability.

Instead, use GitHub Security Advisories:

1. Go to the repository page on GitHub
2. Click the **Security** tab
3. Click **Report a vulnerability**

### What to Include

When reporting, please include as much of the following as possible:

* A description of the vulnerability and its potential impact
* Steps to reproduce (proof-of-concept code if available)
* Affected versions and any known mitigations
* Any relevant logs, stack traces, or screenshots

## Response Timeline

We aim to:

* **Acknowledge** receipt within **48 hours**
* Provide a **status update** within **7 days**
* Work with the reporter on a coordinated disclosure when a fix is available

## Scope

This policy covers:

* Vulnerabilities in the `mxevents` library source code and its intended usage
* Vulnerabilities in the included classification/enrichment logic

This policy does **not** cover:

* Vulnerabilities caused by **misuse** or insecure configuration by consumers
* Vulnerabilities in third-party services outside of this repository
